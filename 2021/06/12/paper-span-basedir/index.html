Joint entity recognition and relation extraction as a multi-head selection problem

ECAI 2018

Motivation

  • State-of-the-art models for joint entity recognition and relation extraction strongly rely on external natural language processing (NLP) tools .In this paper , we proposed a joint neural model without the need of any manually extracted features .Specifically, we model the entity recognition task using CRF layer and the relation extraction task as a multi-head selection problem [-2018].
  • Current joint extraction model cannot have a principled way to deal with multiple relation .

Methods

The outputs for each token (e.g., Smith) are twofold: (i) an entity recognition label (e.g., I-PER, denoting the token is inside a named entity of type PER) and (ii) a set of tuples comprising the head tokens of the entity and the types of relations between them (e.g., {(Center, Works for), (Atlanta, Lives in)})

TPLinker : Single-stage Joint Extraction of Entities and Relations Through Token Pair Linking

  • conling 2020

Problem

  • overlapping relations with shared entities : 在实体关系联合抽取任务之中，不仅仅存在实体重叠的问题，更存在关系重叠的问题，即一个实体作为头实体和不同的尾实体之中存在不同的二元关系，如果采用传统的 BIO的标注的方式，那么头实体只能够选择一组关系进行标注。

  • EntityPairOverlap(EPO) and EntityPairOverlap(SEO)

  • exposure bias :暴漏偏差：指在训练阶段是gold实体输入进行关系预测，而在推断阶段是上一步的预测实体输入进行关系判断；导致训练和推断存在不一致

  • cascading errors ： 误差累计问题，

Motivation

  • TPLinker is to answer three Yes/No pseudo questions:
    • Whether p1 and p2 are the start and end positions of the same entity respectively?
    • Whether p1 and p2 are the start positions of two entities with r relation respectively?
    • Whether p1 and p2 are the end positions of two entities with r relation respectively?

Experiments

  • SciERC: SciERC [20] is derived from 500 abstracts of AI pa- pers. The dataset includes six scientific entity (Task, Method, Met- ric, Material, Other-Scientific-Term, Generic) and seven relation types (Compare, Conjunction, Evaluate-For, Used-For, Feature- Of, Part-Of, Hyponym-Of) in a total of 2, 687 sentences. We use the same train (1, 861 sentences), validation (275 sentences) and test (551) split as in [20].
  • We use the Adam Opti- mizer with a linear warmup and linear decay learning rate schedule and a peak learning rate of 5e−5, a dropout before the entity and relation classifier with a rate of 0.1 (both according to [8]), a batch size ofB=2, and width embeddings w of 25 dimensions. No further optimizations were conducted on those parameters. We choose the number of epochs (20), the relation filtering threshold (α = 0.4), as well as the number of negative entity and relation samples per sen- tence (Ne=Nr=100) based on the CoNLL04 development set. We do not specifically tune our model for the other two datasets but use the same hyperparameters instead

  • Most existing joint neural models for Information Extraction (IE) use local task-specific classifiers to predict labels for individual instances (e.g., trigger, relation) regardless of their interactions.
  • ONEIE

Motivation

  • Figure shows a real example where the local argument role classifier predicts a redundant PERSON edge. The model should be able to avoid such mistakes if it is capable of learning and leveraging the fact that it is unusual for an ELECT event to have two PERSON arguments.

  • USING separate taggers to extract entity mentions and event triggers. Note that do not use types predicted by the taggers. Instead, making a joint decision for all knowledge elements at the decoding stage to prevent error propagation and utilize their interactions to improve the prediction of node type.
